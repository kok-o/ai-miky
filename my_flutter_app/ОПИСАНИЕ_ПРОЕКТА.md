    # Полное описание проекта Miku — AI-ассистент (курсовая работа)

    ## 1. Общая информация

    **Название приложения:** Miku  
    **Тип:** Мобильное / веб-приложение — персональный AI-ассистент  
    **Платформы:** Android, iOS, Web, Windows, Linux, macOS  
    **Язык разработки:** Dart  
    **Фреймворк:** Flutter  
    **Версия:** 1.0.0+1  
    **SDK:** Dart ^3.9.2  

    ---

    ## 2. Цель и задачи проекта

    **Цель:** Разработка кроссплатформенного приложения с чатом на основе OpenAI API, аутентификацией и профилем пользователя.

    **Основные задачи:**
    - Интеграция с OpenAI API для диалога с ИИ
    - Регистрация и вход пользователей (Firebase Auth)
    - Хранение истории чата и профиля в облаке (Firestore)
    - Многоязычность (русский, казахский, английский)
    - Настройки приложения (тема, модель ИИ, язык)
    - Профиль пользователя со статистикой и фото

    │   └── app_localizations*.dart # Сгенерированные классы локализации

    │   └── message.dart         # Модель сообщения чата

    │   ├── auth_screen.dart     # Экран входа/регистрации
    │   ├── home_screen.dart     # Главный экран
    │   ├── chat_screen.dart     # Экран чата с ИИ
    │   ├── profile_screen.dart  # Профиль пользователя
    │   └── settings_screen.dart # Настройки
    ├── services/
    │   ├── ai_service.dart       # Работа с OpenAI API
    │   ├── firestore_service.dart # Работа с Firestore
    │   └── storage_service.dart # Загрузка фото в Firebase Storage
    ├── state/
    │   └── app_state.dart       # Глобальное состояние (Provider)
    └── widgets/
        ├── app_bar_custom.dart  # Кастомный AppBar с градиентом
        ├── bottom_nav.dart      # Нижняя навигация

        └── ai_cursor.dart       # Анимированный курсор/логотип на главной

    ### 4.2 Паттерны и подходы

    - **State management:** Provider (ChangeNotifier) — один глобальный `AppState`
    - **Навигация:** BottomNavigationBar + условный экран входа
    - **Локализация:** flutter_localizations + ARB-файлы + intl
    - **Безопасность:** API-ключ в `.env`, не коммитится в репозиторий

    ## 5. Функциональность по экранам

    ### 5.1 Экран авторизации (AuthScreen)

    - **Вход** по email и паролю (Firebase Auth)
    - **Регистрация** нового пользователя
    - Переключение «Войти» / «Зарегистрироваться»
    - Поле пароля с кнопкой **показать/скрыть пароль**
    - Выбор **языка** (русский, казахский, английский) через иконку в AppBar
    - Локализованные сообщения об ошибках (слабый пароль, email занят, неверный пароль и т.д.)
    - Проверка `mounted` перед `setState` после асинхронных операций

    ### 5.2 Главный экран (HomeScreen)

    - Приветствие «Привет, я Miku!» и краткое описание
    - Виджет **AiCursor** — анимированный круг с эффектом «дыхания» и наведения
    - Кнопка **«Начать общение»** — переход к чату
    - Кнопка **«Войти или зарегистрироваться»** для неавторизованных пользователей
    - Все тексты локализованы

    ### 5.3 Чат (ChatScreen)

    - **Отправка сообщений** пользователя в OpenAI API
    - **Сохранение** сообщений пользователя и ответов ИИ в Firestore (коллекция `users/{userId}/messages`)
    - **Отображение** истории в реальном времени (StreamBuilder)
    - Выбор модели ИИ из глобальных настроек (gpt-3.5-turbo, gpt-4o-mini, gpt-4o)
    - Кнопка **«Очистить чат»** — удаление всех сообщений пользователя в Firestore
    - Индикатор загрузки при ожидании ответа ИИ
    - Разделение сообщений пользователя и ИИ (MessageBubble: цвет, выравнивание)
    - Подсветка сообщений-ошибок (по эмодзи в тексте)
    - Требуется авторизация; без входа показывается сообщение об ошибке

    ### 5.4 Профиль (ProfileScreen)

    - **Аватар:** фото из галереи/камеры или иконка по умолчанию
    - **Загрузка фото:** ImagePicker → байты (Uint8List) → Firebase Storage → URL сохраняется в Firestore
    - **Удаление фото** с подтверждением
    - Поля **«Имя»** и **«О себе»** с сохранением в Firestore и SharedPreferences
    - Кнопка **«Сохранить профиль»**
    - **Статистика:** количество сообщений пользователя, «дней подряд» (streak), выбранная модель ИИ
    - Переключатель **темы** (светлая/тёмная)
    - Выбор **модели ИИ** через bottom sheet
    - Ссылка на вход, если пользователь не авторизован
    - Кнопка **«Выйти»** с отображением email
    - Все подписи и сообщения локализованы

    ### 5.5 Настройки (SettingsScreen)

    - Переключатель **тёмной темы**
    - Выбор **языка** (русский, казахский, английский) с отображением текущего
    - Выбор **модели ИИ** (gpt-3.5-turbo, gpt-4o-mini, gpt-4o)
    - Блок «О приложении» с кратким описанием

    ## 6. Сервисы (логика приложения)

    ### 6.1 AiService (ai_service.dart)

    - **Метод:** `sendMessage(String message, {required String model})`
    - Отправка POST-запроса на `https://api.openai.com/v1/chat/completions`
    - API-ключ берётся из `dotenv.env['OPENAI_API_KEY']`
    - Ограничение частоты запросов (минимум 1 с между запросами)
    - Обработка ответов: 200 — текст ответа, 429/401/403/500 — сообщения об ошибках
    - Обработка сетевых и прочих ошибок без использования dart:io (совместимость с веб)
    - Максимальная длина ответа: 500 токенов

    ### 6.2 FirestoreService (firestore_service.dart)

    - **Сообщения:** `saveMessage`, `getMessages` (stream), `clearChat`
    - **Подсчёт сообщений пользователя:** `getUserMessageCount`, `getUserMessageCountStream` (где `isUser == true`)
    - **Дни подряд:** `updateConsecutiveDays`, `getConsecutiveDays`, `getConsecutiveDaysStream` (логика по дате последнего входа)
    - **Профиль:** `saveProfilePhotoUrl`, `getProfilePhotoUrl`, `getProfilePhotoUrlStream`
    - **Профиль пользователя:** `updateUserProfile(displayName, bio)`, `getUserProfile`
    - Коллекции: `users/{userId}` (документ пользователя), `users/{userId}/messages` (сообщения чата)

    ### 6.3 StorageService (storage_service.dart)

    - **Загрузка фото:** `uploadProfilePhoto(userId, Uint8List imageBytes)` — putData в Firebase Storage, путь `profile_photos/{userId}.jpg`
    - **Удаление фото:** `deleteProfilePhoto(userId)`
    - Реализация через `Uint8List` для совместимости с веб и мобильными платформами

    ## 7. Состояние приложения (AppState)

    **Класс:** `AppState extends ChangeNotifier`

    **Хранимые данные:**
    - Тема (светлая/тёмная), выбранная модель ИИ
    - Имя, «О себе», цвет аватара (локально + при наличии — Firestore)
    - Язык приложения (Locale)
    - Счётчики: количество сообщений пользователя, дни подряд
    - URL фото профиля
    - Текущий пользователь Firebase Auth

    **Методы:**
    - `load()` — загрузка из SharedPreferences, подписка на authStateChanges, инициализация статистики пользователя
    - `setThemeMode`, `setSelectedModel`, `setDisplayName`, `setBio`, `setAvatarColor`, `setLocale`
    - `setProfilePhoto`, `uploadProfilePhoto(Uint8List)`
    - `register`, `login` (с опциональной передачей l10n для локализованных ошибок), `logout`
    - Внутренний `_initializeUserStats(userId)` — обновление дней подряд, загрузка профиля и фото, подписка на стримы Firestore

    **Персистентность:** тема, модель, имя, био, цвет аватара, язык — в SharedPreferences; профиль и статистика — в Firestore.

    ## 8. Модели данных

    ### Message (models/message.dart)

    - `text` — текст сообщения
    - `isUser` — true для пользователя, false для ИИ
    - `timestamp` — время (DateTime)
    - `toMap()` / `fromMap()` — сериализация для Firestore

    ### Структура в Firestore

    - `users/{userId}` — документ: displayName, bio, profilePhotoUrl, lastLoginDate, consecutiveDays
    - `users/{userId}/messages` — коллекция документов с полями text, isUser, timestamp

    ## 9. Локализация (l10n)

    - **Языки:** русский (ru), казахский (kk), английский (en)
    - **Файлы:** `app_ru.arb` (шаблон), `app_kk.arb`, `app_en.arb`
    - **Ключи:** охватывают все экраны (авторизация, профиль, чат, настройки, общие фразы, ошибки)
    - **Переключение:** в настройках и на экране авторизации
    - **Хранение выбора:** SharedPreferences, ключ `app_language`
    - В MaterialApp заданы: `locale`, `localizationsDelegates`, `supportedLocales`

    ## 10. Виджеты

    - **AppBarCustom:** AppBar с градиентным заголовком (фиолетовый–голубой), без тени по умолчанию
    - **BottomNav:** 4 пункта — Главная, Чат, Профиль, Настройки (фиксированные подписи)
    - **MessageBubble:** контейнер сообщения с выравниванием и цветом в зависимости от isUser и isError
    - **AiCursor:** анимированный круг с градиентом, «дыханием», реакцией на наведение и нажатие (используется на главном экране)
